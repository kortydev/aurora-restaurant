## 🌟 Ключевые возможности проекта (Core Features)

### 📱 Клиентская часть (Frontend & UX):
* **Адаптивный дизайн в стиле темного минимализма**: Сетка полностью перестроена под мобильные телефоны, планшеты и широкие 4K-мониторы.
* **Интерактивное меню**: Плавное мгновенное переключение гастрономических категорий без перезагрузки страниц.
* **Доступная мобильная навигация**: Интегрировано плавное мобильное бургер-меню для удобного перемещения по разделам на смартфонах.
* **Интерактивная карта**: Настоящая интерактивная Яндекс.Карта с наложенным стильным темно-сером CSS-фильтром.
* **Безопасная страница успеха (`/reservation/success`)**: Выделенный роут подтверждения резерва, выводящий персональные детали брони (Имя, Дата, Время, Количество гостей). Защищен от потери состояния при обновлении страницы.

### 🛡️ Безопасность и Валидация форм:
* **Автоматическая маска телефона**: Нативный обработчик ввода форматирует номер в стандарт `+7 (XXX) ...` прямо в процессе ввода.
* **Строгий анти-спам валидатор**: Блокировка отправки фейковых номеров с повторяющимися цифрами (например, `9999999999` или `0000000000`), проверка полноты маски и защита от выбора прошедших дат на календаре.
* **Защита от double-submit запросов**: Кнопка бронирования блокируется (`disabled`) и меняет статус во время отправки запроса к API, исключая двойную запись в базу данных.

### ⚙️ Панель администратора (Admin Control Panel):
* **Авторизация (Authentication Gate)**: Роут `/admin/dashboard` защищен сессией авторизации (`localStorage`). Прямой доступ без ввода демо-пароля блокируется и перенаправляет на страницу логина.
* **Гибкая фильтрация записей**: Фильтр по статусам («Все», «Новые», «Подтвержденные», «Отмененные») и календарный поиск броней на конкретную дату.
* **Постраничный вывод (Pagination)**: Ограничение вывода до 8 броней на страницу, предотвращающее появление бесконечного списка.
* **Управление статусом на лету (CRUD PATCH)**: Изменение статуса записи («Новая» / «Подтверждена» / «Отменена») в один клик с моментальным обновлением в базе данных Supabase без перезагрузки страницы.
* **Предупреждающая демо-плашка**: Элегантный баннер, информирующий работодателей и клиентов о демонстрационной версии панели.

## 🛠️ Стек технологий (Tech Stack)

### Frontend:
* **React 18** (Vite)
* **Tailwind CSS** (Стилизация)
* **React Router v6** (Клиентский роутинг)
* **Axios** (Сетевые запросы к API)
* **Lucide React** (Премиальный набор SVG-иконок)

### Backend & Database:
* **Node.js + Express** (REST API)
* **Supabase** (Облачная реляционная база данных PostgreSQL)
* **@supabase/supabase-js** (Официальный SDK клиент для БД)
* **Cors & Dotenv** (CORS-политики и переменные окружения)

## 📊 Структура базы данных (Database Schema)

Таблица `bookings` в PostgreSQL (Supabase) спроектирована по следующей SQL-схеме:

```sql
create table bookings (
  id uuid default gen_random_uuid() primary key,
  name text not null,
  phone text not null,
  booking_date date not null,
  booking_time time not null,
  guests integer not null,
  status text not null default 'new',
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);
```

## 💻 Локальный запуск и установка (Installation & Setup)

Для запуска проекта на локальном компьютере выполните следующие шаги:

### 1. Клонирование репозитория:
```bash
git clone https://github.com/kortydev/aurora-restaurant.git
cd aurora-restaurant
```

### 2. Настройка Бэкенда (Node.js):
```bash
cd backend
npm install
```
Создайте файл `.env` в папке `backend/` и заполните своими ключами:
```env
PORT=5000
SUPABASE_URL=your_supabase_url
SUPABASE_ANON_KEY=your_supabase_anon_key
```

Запуск сервера бэкенда в режиме разработки:
```bash
npm run dev
```

### 3. Настройка Фронтенда (React):
```bash
cd ../frontend
npm install
npm run dev
```
Откройте в браузере: [http://localhost:5173](http://localhost:5173)
